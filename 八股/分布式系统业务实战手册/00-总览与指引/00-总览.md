# 分布式系统业务实战手册 · 总览

> **定位**：面向真实业务场景的分布式系统设计知识库。不拘泥于特定语言或框架，聚焦"为什么这样做""什么时候该这样做""怎么避坑"。
>
> **目标读者**：后端工程师、架构师、面试备战者、技术选型决策者。

## 🎯 学习路径建议

### 📖 推荐阅读顺序（系统学习）

按以下顺序阅读，从基础到进阶，从技术到业务到运维，形成完整闭环：

```
第一阶段：技术基础（打地基）
  01-Redis 业务场景 → 02-MySQL 业务场景
  → 掌握两大核心中间件

第二阶段：架构认知（建框架）
  14-单体与分布式架构选型 → 13-存储引擎深度对比
  → 理解架构 trade-off 和底层存储

第三阶段：一致性核心（啃硬骨头）
  03-数据一致性专题（索引）→ 数据一致性子目录
    01-理论基础 → 02-共识算法 → 03-缓存一致性 → 04-事务与Seata
    → 05-消息主从 → 06-对账补偿 → 07-强一致前沿
    → 08-异步长链路 → 09-异地多活 → 10-场景选型 → 11-时间与时钟
  → 一致性是分布式系统最复杂的问题

第四阶段：设计模式（学套路）
  04-分布式系统核心设计模式
  → ID/CAP/限流/降级/网关/RPC/安全/性能

第五阶段：组合实战（看全貌）
  05-中间件组合架构 → 07-分布式定时任务
  → 中间件怎么组合用

第六阶段：业务系统（落地业务）
  09-电商交易核心 → 10-营销与用户 → 11-内容与搜索 → 12-风控与数据
  → 真实业务怎么设计

第七阶段：运维可观测（保稳定）
  15-链路追踪 → 16-监控告警 → 17-日志体系 → 18-错误处理 → 19-SRE与混沌工程
  → 上线后怎么保稳定

第八阶段：面试冲刺
  06-高频业务场景面试题 → 08-附录（速记卡+Checklist）
  → 面试前过一遍
```

### 路径一：面试备战（突击 2 周）

```
Day 1-3:   01-Redis → 02-MySQL → 13-存储引擎
Day 4-7:   数据一致性子目录（重点）→ 04-设计模式
Day 8-10:  05-组合架构 → 07-定时任务 → 14-架构选型
Day 11-12: 09-12 业务系统 → 15-19 运维
Day 13-14: 06-面试题 → 08-附录速记卡
```

### 路径二：技术选型（项目实战）

```
选型不脱离业务，按场景查阅：

1. 缓存选型 → 01-Redis §17
2. 数据库选型 → 02-MySQL §11
3. 一致性方案选型 → 03 §11 决策树
4. 任务调度选型 → 07 §3
5. 网关/监控/容量 → 04 §11-13
6. 整体架构 → 05
7. 各方案上线前 → 08 §二 落地 Checklist
```

### 路径三：系统学习（深入）

```
按序号 01 → 12 完整阅读，每篇配合参考资料深入。
重点：03（一致性）、04（设计模式）、07（定时任务）、09-12（业务系统）是架构师必修。
```
